# دليل نشر البرنامج (المرحلة 1)

الهدف: تحويل البرنامج لملف تثبيت يحمله الناس، مع تحديث تلقائي —
أي تعديل ترفعه يوصل للكل تلقائياً.

---

## 🔴 خطوة 0 (عاجلة قبل أي شي): غيّر توكن Discord

التوكن القديم كان مكتوباً داخل الكود واعتُبر محروقاً.

1. افتح https://discord.com/developers/applications
2. اختر بوتك → **Bot** → **Reset Token** → انسخ التوكن الجديد.
3. أنشئ ملف اسمه `token.txt` بجوار `run-admin.bat`، والصق التوكن فيه (سطر واحد).
   - هذا الملف **لا يُرفع على GitHub** (محمي في .gitignore).
4. لتشغيل البرنامج مع البوت عندك: شغّل **`run-admin.bat`**.

> ملاحظة: البوت الآن يعمل عندك فقط (المدير)، ولا يُحزم في نسخة المستخدمين إطلاقاً.

---

## خطوة 1: ثبّت أدوات الرفع

- حمّل وثبّت **GitHub Desktop** من https://desktop.github.com
  (يثبّت Git تلقائياً — واجهة رسومية سهلة بدون أوامر).
- سجّل الدخول بحساب GitHub داخل البرنامج.

---

## خطوة 2: عدّل اسم حسابك في الإعدادات

افتح ملف `electron-builder.yml` وغيّر السطر:

```yaml
publish:
  provider: github
  owner: YOUR_GITHUB_USERNAME   # ← اكتب اسم حسابك في GitHub هنا
  repo: gta-fivem-mod-manager
```

---

## خطوة 3: ارفع المشروع على GitHub

1. في GitHub Desktop: **File → Add local repository** واختر مجلد
   `gta-fivem-mod-manager`. لو طلب إنشاء مستودع، اختر **Create a repository**.
2. اضغط **Publish repository**.
3. ⚠️ **مهم جداً: اجعل المستودع Public (عام).**
   - المستودع العام ضروري ليشتغل التحديث التلقائي مجاناً بدون توكنات.
   - آمن الآن: التوكن اتشال، ومجلد المودات (1GB) مستثنى ولن يُرفع.

---

## خطوة 4: أطلق أول إصدار

1. في `package.json` تأكد من رقم النسخة، مثلاً `"version": "0.1.0"`.
2. في GitHub Desktop: من القائمة العلوية **History**، ثم من قائمة الفروع/الوسوم
   أنشئ وسماً (tag) بنفس الرقم مسبوقاً بحرف v — مثال: **`v0.1.0`**.
   - أو أسهل: من موقع GitHub → **Releases → Draft a new release → Create new tag**
     واكتب `v0.1.0` ثم انشره.
3. بمجرد وصول الوسم، **GitHub Actions يبني الـ.exe تلقائياً** (يستغرق ~3-5 دقائق).
   تابع التقدم من تبويب **Actions** في مستودعك.
4. بعد أن ينتهي: افتح تبويب **Releases**، ستجد إصداراً فيه ملف التثبيت `.exe`.
   لو كان الإصدار **Draft (مسودة)**، اضغط **Publish release** لينشر للناس.

---

## خطوة 5: وزّع البرنامج

- أعطِ الناس رابط ملف `.exe` من صفحة Releases.
- عند فتح الملف أول مرة قد يظهر تحذير ويندوز الأزرق
  "Windows protected your PC" (لأنه غير موقّع بشهادة مدفوعة) —
  يضغط المستخدم **More info → Run anyway**. عادي ولا يمنع التثبيت.

---

## كيف أنشر تحديثاً لاحقاً؟

1. عدّل ما تريد في الكود.
2. ارفع رقم النسخة في `package.json` (مثلاً `0.1.0` ← `0.1.1`).
3. Commit + Push من GitHub Desktop.
4. أنشئ وسماً جديداً `v0.1.1` (كما في خطوة 4).
5. GitHub يبني وينشر تلقائياً → **برامج كل المستخدمين تكتشف التحديث،
   تنزّله في الخلفية، وتثبّته عند إغلاق البرنامج.** ✅

---

## المرحلة 2 (لاحقاً): توصيل المودات للجميع

المودات (1GB) لا تُحزم في البرنامج ولا تُرفع في الكود. الخطة:
رفعها كـ **Release assets** على GitHub، وإضافة كود في البرنامج
ينزّلها ويحدّثها عند الطلب، وإعادة توجيه أوامر بوت Discord لتحديث
فهرس مركزي بدل الحالة المحلية. نبدأها بعد نجاح المرحلة 1.
